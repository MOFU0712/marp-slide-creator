# 初回実装 要求内容

## 概要

slide-ai の初回実装。Phase 1（プロトタイプ）として、ローカル実行可能な最小構成を構築する。
Claude API は Next.js API Route 経由で呼び出し、データはセッション中のみメモリ上に保持（永続化なし）。
「骨子生成 → スライド編集 → Marp md エクスポート」の主要フローを一通り動かすことが目標。

---

## スコープ（今回実装するもの）

### Phase 1 対象機能

#### 1. ストーリー設計フェーズ
- [ ] モード選択UI（提案 / 実装報告 / 技術比較 / 調査報告）
- [ ] インタビューフォーム（モード別3〜4問）
- [ ] 素材テキストの入力（ファイルアップロードではなくテキスト貼り付けで代替）
- [ ] Claude API を呼び出して骨子をSSEストリーミングで生成
- [ ] 骨子レビューUI（スライドカード一覧・順序変更・差し戻し・承認）

#### 2. スライド編集フェーズ
- [ ] 3ペインレイアウト（サムネイル一覧 / プレビュー / 編集パネル）
- [ ] テキストタブ（タイトル・本文・スピーカーノートの直接編集）
- [ ] スタイルタブ（レイアウト選択・テーマCSS編集）
- [ ] AIチャットタブ（スライド単位の修正指示、Claude API 呼び出し）
- [ ] @marp-core によるリアルタイムプレビュー

#### 3. 図エディタ
- [ ] 図タイプ選択（5種類）
- [ ] ノード・エッジのテキスト定義
- [ ] SVGキャンバス（ノードのドラッグ移動・クリック選択）
- [ ] プロパティパネル（ラベル・カラー・形状・サイズ編集）
- [ ] 自動整列（横 / 縦）
- [ ] 各図タイプのSVGレンダラ（5種類）

#### 4. エクスポート
- [ ] Marp Markdown（.md）ダウンロード

#### 5. データ管理
- [ ] zustand でセッション中のデータをメモリ上に保持（永続化なし）

---

## スコープ外（今回実装しないもの）

- ローカルファイル/ディレクトリ指定による素材読み込み（Phase 1 はテキスト貼り付けで代替）
- pptx・PDF・HTMLエクスポート（Phase 1 は Marp md のみ）
- データ永続化（セッション終了でデータは破棄される設計）

---

## ユーザーストーリー（Phase 1 対象）

```
As エンジニア
I want 実装ドキュメントのテキストを貼り付けてモードを選び、インタビューに答えると骨子が生成される
So that スライドの構成を考える時間を省ける

As エンジニア
I want 骨子レビュー画面でスライドの順序を変更・差し戻し・承認できる
So that AIの提案を自分のストーリーに合わせて調整できる

As エンジニア
I want スライド編集画面でテキストを直接編集しAIに修正指示を出せる
So that 細かい表現をスムーズに仕上げられる

As エンジニア
I want 図エディタでノードを追加・削除・接続変更し、スタイルをUIで調整できる
So that SVGの微調整のたびにAIに指示を出す手間をなくせる

As エンジニア
I want 完成したスライドをMarp md でダウンロードできる
So that Marp CLI でpptx/PDFに変換したり、Gitで管理したりできる
```

---

## 受け入れ条件

### ストーリー設計フェーズ
- [ ] モードを選択するとインタビューフォームが表示される
- [ ] インタビュー回答と素材テキストを入力してサブミットすると骨子生成が始まる
- [ ] 骨子はSSEストリーミングでスライドカードが順次表示される
- [ ] 骨子レビュー画面でスライドカードのドラッグ&ドロップによる順序変更ができる
- [ ] 「骨子を再生成」ボタンで追加指示を添えて再生成できる
- [ ] 「編集フェーズへ」ボタンでスライド編集画面に遷移する

### スライド編集フェーズ
- [ ] 左サムネイルをクリックするとプレビューが切り替わる
- [ ] テキストタブで編集した内容がプレビューに即時反映される
- [ ] AIチャットタブで指示を送ると対象スライドが更新される
- [ ] スタイルタブでテーマCSSを編集すると全スライドのプレビューに反映される

### 図エディタ
- [ ] 図タイプを選択するとキャンバスに対応する図が表示される
- [ ] ノード定義テキストを編集するとキャンバスが更新される
- [ ] ノードをドラッグして位置を変更できる
- [ ] ノードをクリックするとプロパティパネルが表示され、ラベル・カラー・形状を変更できる
- [ ] 「自動整列」ボタンでノードが整列される
- [ ] 「スライドに挿入」ボタンで図がスライドに反映される

### エクスポート
- [ ] 「Marp出力」ボタンで `.md` ファイルがダウンロードされる
- [ ] ダウンロードしたファイルを Marp CLI で変換するとスライドが生成される

### データ管理
- [ ] セッション中はスライドデータがメモリ上に保持される
- [ ] ページをリロードするとデータは初期化される（意図した動作）

---

## 制約事項

- Claude API キーは環境変数（`.env.local`）に置く
  - `NEXT_PUBLIC_` プレフィックスは使わず、API Routeを経由して呼び出す（Next.js の `/app/api/` を活用）
  - ブラウザには API キーを露出させない
- データはセッション中のみメモリ上に保持し、永続化しない
  - ページリロードでデータは初期化される
  - 作業結果は Marp md エクスポートで保存する運用
- 図のSVGはメモリ上に保存せず、パラメータ（JSON）のみ保持してレンダリングはオンデマンドで行う
