# プロダクト要求定義書

## プロダクトビジョンと目的

エンジニア向けのAI支援スライド作成ツール。
ローカルの素材ドキュメント（ファイルまたはディレクトリ）を指定し、ストーリー構成から図の生成・スタイル調整まで対話的に行えるローカル実行型アプリケーション。

**特徴：**

- 各自がローカル環境で起動して使用
- データベース不要、アプリにデータを残さない
- 素材はローカルファイルシステムから読み込み

**解決する課題：**

- スライドのストーリー設計に時間がかかる（素材はあるが構成に迷う）
- SVG図の微調整のたびにAIに指示を出し直す手間がかかる
- pptxはAIフレンドリーでなく、テキストとして再利用しにくい
- ジュニアエンジニアが一人でスライドを完成させるのが難しい

---

## ターゲットユーザーと課題・ニーズ

### プライマリユーザー

**エンジニア（シニア〜ジュニア）**

- 技術提案・実装報告・技術比較などのスライドを定期的に作成する
- Markdownやコードに慣れており、GUIよりテキスト定義を好む傾向がある
- スライドデザインより内容の質に時間を使いたい

### セカンダリユーザー

**非エンジニア（エンジニアに近い職種）**

- テクニカルライター、プロダクトマネージャーなど
- エンジニア向け資料を作成するが、構成の型が分からない

### ユーザーの主なニーズ

| ニーズ | 現状の問題 | 本ツールでの解決 |
|---|---|---|
| 素材からストーリーを組み立てたい | 構成を一から考える必要がある | モード別テンプレ＋AIインタビューで骨子を自動提案 |
| 図を素早く作りたい | SVGの微調整のたびにトークンを消費する | 図タイプ＋パラメータUIで直接操作、AIはデータのみ返す |
| Marp形式で出力したい | 手書きのMarkdownが必要 | JSON→Marp変換を自動化 |
| pptxにも対応したい | Marp CLIの環境構築が必要 | エクスポート機能で対応 |

## 主要な機能一覧

### 1. ストーリー設計機能

- モード選択（提案 / 実装報告 / 技術比較 / 調査報告）
- AIインタビュー（モード別3〜4問、聴衆・目的・懸念点を収集）
- 素材ドキュメントの指定（ローカルのファイルまたはディレクトリを指定。Markdown・テキスト対応）
- 骨子自動生成（スライド一覧＋各スライドの役割・キーメッセージ）
- 骨子レビュー・差し戻し・再生成

### 2. スライド編集機能

- スライドサムネイル一覧（ドラッグで順序変更）
- テキスト直接編集（タイトル・本文・スピーカーノート）
- AIチャットによる修正指示（スライド単位）
- レイアウト切り替え（タイトル+箇条書き / タイトル+図 / 2カラム / 区切り）
- CSSテーマ設定（全スライドに適用）

### 3. 図エディタ機能

- 図タイプ選択（アーキテクチャ / フロー図 / 棒グラフ / タイムライン / 比較表）
- ノード・エッジのテキスト定義
- ビジュアルエディタ（ノードドラッグ・プロパティ編集）
- 自動整列
- AIによる図パラメータ提案（SVGは生成しない）

### 4. プレビュー機能

- リアルタイムプレビュー（ブラウザ内Marpレンダリング）
- スライドナビゲーション

### 5. エクスポート機能

- Marp Markdown（.md）
- PowerPoint（.pptx）
- PDF
- HTML（ローカルファイル）

## 成功の定義

### 定量指標

- スライド1本の作成時間を従来比50%以下に短縮
- 骨子生成から編集開始まで5分以内
- 図の作成・調整をAIとのやり取りなしで完結できる

### 定性指標

- ジュニアエンジニアが一人でスライドを完成できる
- 「素材からどう話すか」の設計をAIと一緒に考えられる
- 生成したスライドをそのまま発表に使えるクオリティ

---

## ビジネス要件

- 個人・チームがローカルで使用するツールとして開発
- OSSとして公開予定
- APIコスト（Claude API）は各ユーザーが自身のAPIキーを使用

## ユーザーストーリー

```
As エンジニア
I want 実装ドキュメントをアップロードして「実装報告モード」で骨子を自動生成したい
So that スライドの構成を考える時間を省き、内容の質を上げることができる

As エンジニア
I want 図のノードをUIで追加・削除・接続変更したい
So that SVGの微調整のたびにAIに指示を出す手間をなくしたい

As エンジニア
I want 完成したスライドをpptxとMarp mdの両方で出力したい
So that 発表環境に合わせて使い分けられる

As ジュニアエンジニア
I want AIインタビューに答えるだけで骨子を提案してほしい
So that ストーリー設計の型を学びながらスライドを作れる
```

---

## 受け入れ条件

- [ ] ローカルファイル/ディレクトリを指定してモードを選択し、AIインタビューに答えると骨子が生成される
- [ ] 骨子レビュー画面でスライドの順序変更・差し戻し・再生成ができる
- [ ] スライド編集画面でテキスト編集・AI修正・図エディタが動作する
- [ ] 図エディタでノード追加・削除・接続変更・スタイル変更ができる
- [ ] プレビューがリアルタイムで更新される
- [ ] Marp md・pptx・PDFでエクスポートできる
- [ ] セッション中のみデータを保持し、終了時にはデータが残らない

---

## 機能要件

### FR-01 ストーリー設計

- モードは4種類（提案 / 実装報告 / 技術比較 / 調査報告）から選択
- インタビューはモード別に3〜4問、フォーム形式で一括回答
- 素材はローカルのファイルパスまたはディレクトリパスを指定（Markdown・テキスト対応）
- 骨子生成はストリーミングでリアルタイム表示
- 骨子はスライドタイトル・役割・キーメッセージ・推奨図タイプで構成

### FR-02 スライド編集

- スライド数は最大30枚
- テキスト変更はプレビューに即時反映
- AIチャットはスライド単位、会話履歴は最新10往復を保持
- CSSはMarp準拠のCSS記法で記述

### FR-03 図エディタ

- 図タイプは5種類（アーキテクチャ / フロー図 / 棒グラフ / タイムライン / 比較表）
- ノード数は最大20個
- AIはノード・エッジのJSONパラメータのみ返し、SVGは生成しない
- SVGレンダリングはフロントエンドのテンプレートエンジンが担当

### FR-04 エクスポート

- pptx・PDF変換はMarp CLIで処理
- HTMLエクスポートはローカルファイルとして出力

---

## 非機能要件

### パフォーマンス

- 骨子生成：最初のトークンが3秒以内に表示される
- スライドプレビュー更新：操作から500ms以内
- エクスポート（pptx）：30秒以内に完了

### セキュリティ

- Claude APIキーは環境変数または設定ファイルで管理（ユーザーが自身のキーを設定）
- ローカル実行のため、データはユーザーのマシン外に出ない
- セッション終了時にメモリ上のデータは破棄

### データ管理

- アプリにデータを永続化しない（ステートレス）
- 素材ファイルはローカルファイルシステムから読み込み
- エクスポート結果はユーザー指定のローカルパスに出力
